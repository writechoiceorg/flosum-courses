# Lesson 2: Two Different Worlds

Salesforce and Git operate on fundamentally different models. This lesson explains the structural gap that makes standard DevOps tooling the wrong fit.

* [**Git vs. Salesforce**](git-vs-salesforce.md): why Git and Salesforce differ in kind, not just degree, including the topology inversion that makes standard branching fail.
* [**Who Each System Was Designed For**](who-each-system-was-designed-for.md): Git was built for developers; Salesforce was built for admins, and the audience mismatch creates mandatory translation overhead that scales poorly.
* [**Why This Difference Matters**](why-this-difference-matters.md): three consequences of the design gap, topology inversion, absent back-sync path, and compounding drift, that produce real release failures.
