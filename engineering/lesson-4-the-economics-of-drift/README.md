# Lesson 4: The Economics of Drift

The seven mismatches map directly to DORA metrics and measurable operational costs: this lesson provides the formula structure to quantify the architectural tax.

* [**Where the Money Leaks**](where-the-money-leaks.md): five cost categories with DORA metric mappings: failed deployments, rollback hours, release delays, audit risk, and developer burnout.
* [**The Human Translator Tax**](human-translator-tax.md): why the Release Manager role exists, what it costs, and the formula including variable tables and sourced ranges.
* [**The Trust Tax**](trust-tax.md): how pipeline failures erode team confidence, generating slower cadence, more manual gates, and routing around the pipeline.
* [**The Break-Even Calculation**](break-even-calculation.md): the comparison structure and worked example for calculating when a purpose-built platform pays for itself.
* [**The Formal Cost Model**](formal-cost-model.md): the three formulas (Human Translator Tax, Deployment Loss, Compliance Overhead) with full variable tables, sourcing tiers, and documented assumptions.
* [**DORA Metric Mapping**](dora-metric-mapping.md): maps all four DORA metrics to structural causes in the Git-Salesforce architecture, with tier benchmarks for each.
* [**Sensitivity-Range Spreadsheet**](sensitivity-range-spreadsheet.md): identifies which inputs drive the most cost model variance: where to measure precisely and where defaults are acceptable.
